# Как работаем
- **Начало работы:** агент приступает к реализации сразу, без предварительного плана (пользователь предпочитает «Сразу действовать»). Если нужны уточнения – агент задаёт короткие уточняющие вопросы перед выполнением.
- **Проверка результата:** после выполнения агент предоставляет список изменённых файлов и краткое описание того, что сделано. По запросу можно добавить короткие комментарии в сгенерированный код.
- **Итоговый вывод:** в конце ответа должен бытьbrief summary (одно‑две строки) о выполненных изменениях.

# Чего не делать
## Границы
- Не удалять и не переименовывать файлы вне рабочей папки без явного подтверждения пользователя.
- Не выполнять никаких действий с базой данных без прямого указания.
- При изменениях в моделях миграции выполнять только после получения подтверждения.
- Не пушить изменения в репозиторий, не создавать pull‑request без прямого указания пользователя.
- Не менять переменные окружения, конфигурационные файлы (docker‑compose.yml, .env и аналогичные) без подтверждения.

## Анти‑паттерны
- Длинные вводные фразы перед ответом.
- Шаблонные фразы типа «конечно!».
- Повторение того, что только что сделал агент.
- Избыточные объяснения очевидных вещей.

# Формат ответа
- Код в ответе снабжается краткими inline‑комментариями только там, где требуется пояснение.
- Если нужны дополнительные уточнения – агент задаёт короткие вопросы, а не предполагает.
- В завершении ответа обязателен краткий вывод (одно‑две строки) о том, какие файлы изменены и что сделано.
