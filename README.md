## Аутентификация

Приложение использует JWT-аутентификацию для защиты API.

### Эндпоинты аутентификации:

- `POST /auth/register` - регистрация нового пользователя
- `POST /auth/login` - получение JWT-токена
- `GET /auth/me` - получение информации о текущем пользователе

Для доступа к защищенным эндпоинтам (например, `/books/borrows`) необходимо передавать токен в заголовке `Authorization: Bearer <token>`.

## Запуск приложения

Для запуска приложения необходимо:

1. Создать файл .env в соответствии с файлом .env.sample
   - Убедитесь, что вы заменили SECRET_KEY на надежный секретный ключ для production
   - Команда для генерации ключа: `openssl rand -hex 32`
2. Запустить контейнер в докере командой `docker-compose up --build -d`
3. Подключиться к базе данных, используя переменные из файла .env (host - POSTGRES_HOST, port - POSTGRES_PORT)

Документация будет доступна по адресу http://127.0.0.1:8000/docs/
